## 1. 游戏概览

### 核心概念
玩家控制由多个小立方体组成的滚动立方体生物，在无尽的平坦战场上战斗。核心玩法循环：滚动移动 → 战斗敌人 → 吸收掉落的立方体 → 变大变强 → 面对更强的敌人。

### 设计理念
- **简单易懂的机制，深度可挖掘的玩法**
- **核心情感体验**："成长的满足感"
- **零干扰UI**：完全无文字、无HUD，纯粹的游戏体验

### 游戏模式
- **无尽生存**：在越来越难的敌人波次中生存，长得尽可能大
- **程序生成**：动态平衡的敌人生成系统

## 2. 核心机制

### 2.1 移动系统
- **自动滚动**：玩家始终向前滚动，施加持续前向力
- **方向控制**：鼠标控制镜头方向（即前进方向）
- **跳跃**：鼠标左键跳跃
- **快速坠落**：滞空时再次点击左键，快速坠落（造成伤害）

**相机系统**（第三人称固定跟随）：
- 相机在玩家后方固定距离和高度
- 平滑跟随，减少抖动
- 镜头方向 = 前进方向（鼠标控制）
- 参数可配：距离、高度、角度、平滑系数

### 2.2 战斗系统
- **普通碰撞**：无伤害，仅物理碰撞
- **坠落攻击**：快速坠落时碰撞敌人造成伤害
  - 伤害公式：`damage = ceil(attacker.currentCubes / 8)`
  - 玩家和敌人使用相同机制
  - 敌人AI会尝试跳起并快速坠落攻击玩家
- **三棱锥**：鼠标右键发射，对敌人造成减速效果
  - 大小：当前立方体大小的 1/8
  - 减速效果：移动速度减半 + 禁用跳跃能力
  - 禁用跳跃为隐藏机制，玩家无法直接感知
  - 持续时间：3秒（可配）

### 2.3 成长系统
**立方体爆出机制**：
- 每造成 1/8 最大血量的伤害 → 敌人爆出 1 个立方体
- 累积伤害计数器追踪，达到阈值触发爆出
- 当敌人立方体数 < 历史最大值的 1/8 时彻底死亡
- 彻底死亡时爆出所有剩余立方体

**立方体吸收机制**：
- 碰撞掉落物时吸收其 1/2 数值
- 使用缓冲区系统（数据与表现分离）
- 缓冲区累积到 1.0 时添加 1 个立方体到集群

**万有引力修正**：
- 吸收立方体后通过算法计算最佳生长位置
- 使整体形状趋向球体（非物理力，位置重算）

## 3. 数值系统

### 3.1 核心数值

| 数值 | 公式 | 说明 |
|------|------|------|
| 历史最大立方体 | `max(8, 实际历史最大值)` | 最小为8 |
| 当前立方体 | `currentCubes` | 实际持有的立方体数 |
| 血量上限 | `maxHistoricalCubes × 1` | 基于历史最大值 |
| 当前血量 | `currentCubes × 1` | 基于当前值 |
| 攻击力 | `ceil(maxHistoricalCubes / 8)` | 向上取整，最小为1 |

### 3.2 伤害计算

```
单次坠落伤害 = ceil(attacker.currentCubes / 8)

累积伤害计数器：accumulatedDamage
爆出阈值 = maxHealth × 0.125

爆出流程：
1. accumulatedDamage += 单次坠落伤害
2. 当 accumulatedDamage ≥ 爆出阈值时：
   - 移除1个立方体
   - 生成1个掉落物（value = 1）
   - accumulatedDamage -= 爆出阈值

死亡条件：
当 currentCubes < maxHistoricalCubes × 0.125 时彻底死亡
```

### 3.3 吸收系统

```
absorptionBuffer: float

碰撞掉落物（value）时：
absorptionBuffer += value × 0.5

当 absorptionBuffer ≥ 1.0 时：
    cubesToAdd = floor(absorptionBuffer)
    currentCubes += cubesToAdd
    absorptionBuffer -= cubesToAdd
```

### 3.4 敌人生成

```
wave = 1, 2, 3, ...

enemyCubes = 4 + wave × 2
spawnInterval = max(2, 8 - wave × 0.5)

动态平衡调整范围：±30%
```

## 4. 架构设计

### 4.1 技术栈
- **渲染引擎**：Three.js（WebGL）
- **物理引擎**：Ammo.js（WASM，高性能）
- **状态管理**：观察者模式或轻量状态机
- **配置系统**：JSON 驱动，支持热重载

### 4.2 核心架构分层

```
┌─────────────────────────────────────┐
│          UI 层（极简）               │
│    开始界面 / 游戏结束界面            │
└─────────────────────────────────────┘
                 ↑
┌─────────────────────────────────────┐
│         游戏逻辑层                   │
│  - EntityManager                    │
│  - CombatSystem                     │
│  - GrowthSystem                     │
│  - DifficultyBalance                │
└─────────────────────────────────────┘
                 ↑
┌─────────────────────────────────────┐
│          物理层                      │
│  - 刚体模拟                          │
│  - 碰撞检测                          │
│  - 力应用                            │
└─────────────────────────────────────┘
                 ↑
┌─────────────────────────────────────┐
│          渲染层                      │
│  - Three.js 场景管理                 │
│  - 相机、光照、材质                   │
└─────────────────────────────────────┘
                 ↑
┌─────────────────────────────────────┐
│          配置层                      │
│  - JSON 数值配表                     │
└─────────────────────────────────────┘
```

### 4.3 数据流
```
玩家输入 → 移动/攻击逻辑 → 物理模拟 → 碰撞检测 → 战斗计算 → 状态更新 → 渲染反馈
```

### 4.4 关键设计决策
- 使用 ECS（Entity-Component-System）变体，便于实体扩展
- 物理驱动渲染，确保视觉和物理一致
- 所有数值外部化到 JSON 配置文件
- 万有引力修正为算法计算，非物理力
- 使用 Ammo.js WASM 版本，应对海量物理对象

### 4.5 性能优化策略

**掉落物生命周期管理**：
```
策略：数量上限 + 自动销毁组合

参数（可配）：
- 数量上限：200个
- 存在时间：10秒后自动销毁
- 超过上限时：优先销毁最老的掉落物

实现：
- 维护掉落物列表，记录创建时间
- 每帧检查：超时销毁 + 超量销毁
- 确保场景中掉落物总数可控
```

**碰撞检测优化**：
```
策略：空间分区 + 距离剔除组合

空间分区（粗粒度网格）：
- 将场景划分为网格（如 10m × 10m）
- 只检测相邻网格内对象的碰撞
- 大幅减少碰撞检测次数（O(n²) → O(n)）

距离剔除：
- 只检测距离玩家 50m（可配）范围内的对象
- 远离玩家的区域跳过详细计算
- 适合无尽场景

组合效果：
- 先按距离筛选 → 空间分区过滤 → 精确碰撞检测
- 三级过滤，性能最优
```

## 5. 详细组件设计

### 5.1 实体系统

**实体基类（Entity）**：
- `id`: 唯一标识符
- `type`: 玩家/敌人/抛射物/掉落物
- `position`, `rotation`, `velocity`
- `components`: 组件字典

**核心组件**：

| 组件 | 属性 | 说明 |
|------|------|------|
| CubeClusterComponent | cubes, maxCubeCount, currentCubeCount, health | 管理立方体集群 |
| CombatComponent | attackPower, slowEffect | 战斗属性 |
| MovementComponent | rollSpeed, jumpForce, canFastFall | 移动能力 |

### 5.2 战斗系统流程

```
1. 碰撞检测 → onCollision()
2. 判断是否快速坠落状态
3. 如果是：计算伤害 = ceil(currentCubes / 8)
4. 累加到 accumulatedDamage
5. 检查爆出阈值
6. 如达到阈值：爆出1立方体
7. 检查死亡条件
8. 如死亡：爆出所有立方体
```

### 5.3 万有引力修正算法

```
当玩家吸收立方体时：
1. 找到当前立方体集群的重心
2. 计算每个位置到重心的距离
3. 新立方体放置在使整体最接近球体的位置
4. 最小化到理想球体表面的距离方差
```

### 5.4 敌人AI系统

**行为模式**：纯追踪型

```
核心逻辑：
1. 计算到玩家的方向向量
2. 始终朝玩家方向滚动
3. 当距离玩家在一定范围内时，尝试跳跃攻击
4. 滞空时判断时机，尝试快速坠落攻击玩家

参数（可配）：
- 追踪速度：基于敌人立方体数量
- 跳跃触发距离：如 20m
- 快坠落触发时机：滞空时间 > 0.3秒 且玩家在下方
- 攻击冷却时间：防止过于频繁攻击
```

## 6. 视觉反馈与音效设计

### 6.1 视觉反馈系统

**A) 伤害与战斗反馈**
- 被击中时角色闪白光
- 移位时始终有拖尾特效
- 快速坠落的拖尾更强（差异化传达）

**B) 立方体爆出动画**
- 飞出时：略微的弹射效果（润色级别）
- 落地时：反馈特效（尘埃/冲击波）

**C) 减速视觉效果**
- 被三棱锥击中的敌人变色
- 产生粒子效果

**D) 吸收反馈**
- 光效/粒子流向新立方体目标点
- 目标点立方体生成动画：
  - 透明 → 白光 → 转原色固化

### 6.2 音效与屏幕震动

| 事件 | 音效 | 屏幕震动 | 顿帧 |
|------|------|----------|------|
| 跳起正常坠落 | ✓ | - | - |
| 跳起加速坠落 | ✓ | 轻微 | - |
| 加速坠落撞击敌人 | ✓ | 普通 | 1帧 |
| 受到攻击 | ✓ | 普通 | 1帧 |
| 杀死敌人 | ✓ | 略强 | 2帧 |
| 被杀死（死亡） | ✓ | 强 | 3帧 |

**震动强度分级**：无 < 轻微 < 普通 < 略强 < 强

## 7. 游戏循环与状态

### 7.1 状态机

```
MENU → PLAYING (点击开始)
PLAYING → GAME_OVER (玩家死亡)
GAME_OVER → MENU (自动，约2秒后)
```

### 7.2 主循环

**每帧流程**：
1. 输入处理（鼠标移动、点击）
2. 物理更新（固定时间步长 60fps）
3. 游戏逻辑更新
   - 敌人生成（动态平衡）
   - 战斗计算
   - 立方体吸收检测
   - 万有引力修正
4. 渲染更新
   - 同步物理位置到渲染对象
   - 更新相机跟随

### 7.3 动态平衡系统

- 追踪玩家最近 30 秒的击杀速度
- 根据表现调整敌人生成频率和强度
- 目标：维持"有挑战但可获胜"的平衡点

## 8. UI 设计（极简版）

### 8.1 设计原则
- **零干扰体验**：无文字、无HUD、无菜单
- 所有信息通过游戏本身传达

### 8.2 界面流程

**开始界面**：
- 全黑背景或简单渐变
- 点击任意位置 → 直接进入游戏

**游戏中**：
- 完全无UI
- 玩家通过视觉反馈感知状态

**游戏结束**：
- 屏幕淡出到黑色
- 自动返回开始界面（约2秒后）

### 8.3 操作
- 移动：鼠标控制方向
- 左键：跳跃 / 滞空时快速坠落
- 右键：发射三棱锥（减速敌人）
